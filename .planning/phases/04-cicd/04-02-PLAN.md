PSGallery authentication:
- Secret name: `NUGET_API_KEY` (must be set in GitHub repo Settings → Secrets and variables → Actions)
- Injected as env var in the publish step — never reference `secrets.*` directly in `run:` scripts
- `Publish-Module` cmdlet is built into PowerShellGet (no install needed on windows-latest)

Publish trigger (Release, NOT tag push):
- `on: release: types: [published]` → fires only when human explicitly publishes a draft release
- Safer than `on: push: tags: ['v*']` which fires immediately on any matching tag push
- The checkout at release time resolves to the tagged commit SHA

Do NOT add a test step in publish.yml. The release is only created after CI has already passed on main (CICD-01 gate). Adding tests here is redundant overhead.

```yaml
name: Publish to PSGallery

on:
  release:
    types: [published]

jobs:
  publish:
    name: Publish Module
    runs-on: windows-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Publish to PSGallery
        shell: pwsh
        env:
          NUGET_API_KEY: ${{ secrets.NUGET_API_KEY }}
        run: |
          Publish-Module -Path ".\MDEValidator" -NuGetApiKey $env:NUGET_API_KEY -Verbose
```

Security notes (do not deviate from these):
- NUGET_API_KEY is injected via `env:` block and accessed as `$env:NUGET_API_KEY` — never reference `${{ secrets.NUGET_API_KEY }}` directly inside a `run:` script
