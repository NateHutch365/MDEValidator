<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Two GitHub Actions workflows created locally:
- `.github/workflows/ci.yml` — runs Pester tests, uploads coverage, lints with PSScriptAnalyzer on every push/PR to main
- `.github/workflows/publish.yml` — publishes to PSGallery when a GitHub Release is published

The workflows are syntactically verified and the local test suite is green. CI cannot actually run until the code is pushed to a GitHub repository and the required secret is configured.
  </what-built>
  <how-to-verify>
**Step 1 — Push to GitHub and trigger CI:**
1. Ensure the repo has a GitHub remote: `git remote -v`
2. If no remote exists, create a repo at https://github.com/new, then: `git remote add origin https://github.com/YOUR_USERNAME/MDEValidator.git`
3. Push the current branch: `git push -u origin main` (or your branch name + open a PR)
4. Visit the repository's **Actions** tab on GitHub
5. Confirm the **CI** workflow appears and its job ("Test and Lint") completes with a green ✓
6. Confirm artifacts `test-results` and `coverage-report` are visible on the completed workflow run

**Step 2 — Verify PSScriptAnalyzer lint gate (optional, can confirm via Step 1 run):**
No separate action needed — the CI job runs lint as the final step. A green CI run confirms lint passed.

**Step 3 — Configure NUGET_API_KEY secret (required before publish workflow can succeed):**
1. Sign in at https://www.powershellgallery.com
2. Click your username → **API Keys** → Create a new key
3. Scope: "Push new packages and package versions" — restrict to module name `MDEValidator` if the UI allows
4. Copy the key value (shown only once)
5. In GitHub: repository **Settings** → **Secrets and variables** → **Actions** → **New repository secret**
6. Name: `NUGET_API_KEY`, Value: (paste the key)
7. Save

**Step 4 — Verify publish workflow (optional — do this when ready to publish v1.0.0):**
1. In GitHub: **Releases** → **Draft a new release**
2. Tag version: `v1.0.0`, target: `main`, title: `v1.0.0 — Initial release`
3. Add release notes → click **Publish release**
4. Visit **Actions** tab → confirm **Publish to PSGallery** workflow runs and succeeds
5. Visit https://www.powershellgallery.com/packages/MDEValidator — confirm module is listed

**Minimum to call CI/CD Phase complete:** Steps 1 confirms CI is live. Step 3 unblocks publish. Step 4 is optional until you're ready to publish.
  </how-to-verify>
