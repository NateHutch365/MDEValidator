## Objective
Create a test file for each of the 45 public functions by reading the list of public function files and generating a test file stub using the template from Task 3 of Plan 02-01.

## Approach
Create a PowerShell script that:
1. Lists all public functions from `MDEValidator/Public/*.ps1`
2. For each function, generates a test file using the template structure, with:
   - Function name in Describe block
   - Pass path context placeholder
   - Fail path context placeholder
   - Basic template assertions

The generated test files do not need complete implementations yet (that's Task 2); they should be valid Pester structure that can be discovered and run, but may have placeholder assertions.

## Implementation
Write a script (can be inline in this task execution) that:

```powershell
# Get list of public function files
\$publicFunctions = Get-ChildItem './MDEValidator/Public/*.ps1' | 
    Select-Object -ExpandProperty BaseName | 
    Sort-Object

# Template for each test file
\$template = @'
BeforeAll {
    . \$PSScriptRoot/../Helpers/TestBootstrap.ps1
    Initialize-MDEValidatorTest
    
    . \$PSScriptRoot/../Helpers/MockBuilders.ps1
}

Describe '{FunctionName}' {
    Context 'Pass path' {
        It 'returns Pass when validation succeeds' {
            # TODO: Add mock setup and assertion
            \$result = {FunctionName}
            \$result.Status | Should -Be 'Pass'
        }
    }
    
    Context 'Fail path' {
        It 'returns Fail when validation fails' {
            # TODO: Add mock setup and assertion
            \$result = {FunctionName}
            \$result.Status | Should -Be 'Fail'
        }
    }
}
'@

# Generate test file for each function
foreach (\$funcName in \$publicFunctions) {
    \$testContent = \$template -replace '{FunctionName}', \$funcName
    \$testPath = "./Tests/Public/\$funcName.Tests.ps1"
    \$testContent | Out-File -FilePath \$testPath -Encoding UTF8 -Force
}

Write-Host "Generated \$(\$publicFunctions.Count) test file stubs"
```

All 45 test files will be created in Tests/Public/ with the naming pattern `<FunctionName>.Tests.ps1`.

## Objective
Replace placeholder assertions with real mocking and pass/fail assertions for the subset of public functions that primarily use Defender and Service cmdlets (about half of the 45 files).

## Approach
For each targeted function in this batch, add:
1. Mock dependencies based on implementation (`Get-MpPreference`, `Get-MpComputerStatus`, `Get-Service`)
2. Pass path: Function returns Pass when mocked dependencies have expected "good" values
3. Fail path: Function returns Fail when mocked dependencies have expected "bad" values
4. `Should -Invoke` assertions to verify each expected external dependency call

## Implementation Strategy
Since there are 45 functions, the implementation should be informed by:
1. Reading each function's source code to identify which external dependencies it calls
2. Understanding what constitutes a "pass" vs "fail" for that function
3. Creating test cases that exercise both paths

For example:
- `Test-MDEServiceStatus` checks if WinDefend service is Running and Automatic
  - Pass: Mock Get-Service to return Status='Running', StartType='Automatic'
  - Fail: Mock Get-Service to return Status='Stopped'

- `Test-MDECloudProtection` checks if cloud protection is enabled
  - Pass: Mock Get-MpPreference to return DisableCloudProtection=$false
  - Fail: Mock Get-MpPreference to return DisableCloudProtection=$true

## Recommended Implementation
For each test file, analyze the corresponding function file and:
1. Identify which cmdlets it calls from TEST-02 boundary list (Get-MpPreference, Get-MpComputerStatus, Get-Service, Get-ItemProperty)
2. Determine pass/fail input values based on the function's logic
3. Add Mock statements for those cmdlets using `-ModuleName MDEValidator`
4. Add assertions that check:
   - Return value matches expected status (Pass/Fail)
   - Should -Invoke verifies the mocked cmdlet was called

Example pattern for Test-MDEServiceStatus.Tests.ps1:

```powershell
BeforeAll {
    . \$PSScriptRoot/../Helpers/TestBootstrap.ps1
    Initialize-MDEValidatorTest
    
    . \$PSScriptRoot/../Helpers/MockBuilders.ps1
}

Describe 'Test-MDEServiceStatus' {
    Context 'Pass path' {
        It 'returns Pass when WinDefend is running and automatic' {
            Mock Get-Service -ModuleName MDEValidator {
                [pscustomobject]@{ Name = 'WinDefend'; Status = 'Running'; StartType = 'Automatic' }
            }
            
            \$result = Test-MDEServiceStatus
            
            \$result.Status | Should -Be 'Pass'
            Should -Invoke Get-Service -ModuleName MDEValidator -Times 1
        }
    }
    
    Context 'Fail path' {
        It 'returns Fail when WinDefend is stopped' {
            Mock Get-Service -ModuleName MDEValidator {
                [pscustomobject]@{ Name = 'WinDefend'; Status = 'Stopped'; StartType = 'Automatic' }
            }
            
            \$result = Test-MDEServiceStatus
            
            \$result.Status | Should -Be 'Fail'
            Should -Invoke Get-Service -ModuleName MDEValidator -Times 1
        }
    }
}
```

## Notes
- This task intentionally covers only dependency families A/B to keep scope atomic and reviewable.
- Family A: Defender preference/status functions use `Get-MpPreference` and/or `Get-MpComputerStatus` mocks.
- Family B: Service status functions use `Get-Service` mocks.
- Each test file should be self-contained (no execution order dependencies).
- All mocks use `-ModuleName MDEValidator` to inject into module scope.
- Pass/Fail logic should match the actual function implementation.
- Both scenarios in each function should exercise at least one external call boundary.
- Tests should not make live calls to Defender, services, or registry.
